# Customer Profile

## Primary Personas

### 1. 혼자 주말을 보내는 직장인

**Demographics**

- 연령: 28~39세
- 거주지: 서울 또는 수도권
- 직업: 사무직, IT, 금융, 공공기관, 프리랜서
- 생활 패턴: 평일에는 야근과 회식이 잦고, 주말에는 회복과 가벼운 외출을 원함
- 디지털 사용성: 지도, 날씨, 포털 검색, SNS 저장 기능에 익숙함

**Goals**

- 혼자 가도 어색하지 않은 장소를 찾고 싶다.
- 너무 멀지 않고 부담 없는 활동을 하고 싶다.
- 주말을 의미 있게 보냈다는 느낌을 얻고 싶다.
- 날씨가 나쁘면 실내에서 조용히 시간을 보내고 싶다.

**Frustrations**

- “혼자 가기 좋은 곳”이라는 검색 결과가 너무 광고성이다.
- 전시, 도서관, 카페, 산책 장소를 따로 검색해야 한다.
- 막상 가면 사람이 너무 많거나 분위기가 맞지 않는다.
- 무료라고 되어 있어도 실제로는 예약이나 유료 부대비용이 있다.
- 날씨가 더우면 밖에 나가는 것 자체가 부담스럽다.

**Decision Factors**

- 혼자 방문해도 자연스러운 분위기
- 이동 시간 40분 이하
- 실내 여부
- 혼잡도
- 무료 또는 1만원 이하 비용
- 운영 시간과 즉시 방문 가능성

**Preferred Activities**

- 무료 전시
- 박물관, 미술관
- 도서관 문화 프로그램
- 조용한 산책로
- 독립서점
- 문화센터 강연
- 실내 체험 프로그램

**Weather Sensitivity**

- 폭염, 한파, 비에 민감하다.
- 불쾌지수가 높으면 실외 활동을 피한다.
- 맑고 선선한 날에는 산책과 야외 행사를 선호한다.

**Transportation Preference**

- 대중교통 선호
- 환승이 많으면 포기 가능성이 높다.
- 도보 15분 이상은 날씨가 좋을 때만 허용한다.
- 택시는 특별한 경우에만 사용한다.

### 2. 데이트 장소를 찾는 커플

**Demographics**

- 연령: 23~38세
- 관계: 연인, 썸, 부부
- 거주지: 서울 내 다른 자치구 또는 수도권
- 생활 패턴: 평일 저녁 또는 주말 오후 데이트를 자주 계획
- 디지털 사용성: 인스타그램, 지도 저장, 블로그 리뷰, 예약 앱 사용에 익숙함

**Goals**

- 날씨와 분위기에 맞는 데이트 장소를 찾고 싶다.
- 상대방이 좋아할 만한 실패 확률 낮은 선택지를 원한다.
- 식사 전후로 연결 가능한 활동을 찾고 싶다.
- 비가 오거나 너무 더울 때 대체안을 원한다.

**Frustrations**

- 검색 결과가 유명한 장소 위주라 너무 붐빈다.
- SNS에서 본 장소는 실제로 가면 대기가 길다.
- 한 명은 실내를 원하고 다른 한 명은 야외를 원한다.
- 날씨가 갑자기 바뀌면 계획이 무너진다.
- 무료 행사인지, 예약이 필요한지 바로 알기 어렵다.

**Decision Factors**

- 분위기
- 날씨 적합성
- 이동 동선
- 대기 시간
- 비용
- 사진을 남기기 좋은지
- 식사/카페와 연결 가능한지

**Preferred Activities**

- 전시
- 야외 피크닉
- 한강 산책
- 공연
- 팝업스토어
- 마켓, 축제
- 실내 체험
- 야경 명소

**Weather Sensitivity**

- 비가 오면 야외 데이트 만족도가 급감한다.
- 폭염이면 실내 전시, 쇼핑몰, 박물관을 선호한다.
- 선선한 날에는 한강, 공원, 야외 축제를 선호한다.

**Transportation Preference**

- 지하철 접근성 중요
- 두 사람의 중간 지점 선호
- 주말에는 주차 난이도를 싫어한다.
- 이동 시간이 1시간을 넘으면 매력도가 급격히 낮아진다.

### 3. 아이와 외출하려는 부모

**Demographics**

- 연령: 32~48세
- 가족 구성: 영유아, 초등학생 자녀가 있는 부모
- 거주지: 서울 또는 수도권
- 생활 패턴: 주말 오전~오후 가족 외출 계획
- 디지털 사용성: 맘카페, 지도 리뷰, 공공시설 홈페이지, 예약 시스템 사용

**Goals**

- 아이가 즐길 수 있고 안전한 장소를 찾고 싶다.
- 비용 부담이 적은 공공 프로그램을 찾고 싶다.
- 비, 폭염, 미세먼지에도 갈 수 있는 실내 장소를 원한다.
- 이동과 대기 스트레스를 줄이고 싶다.

**Frustrations**

- 아이와 갈 만한 장소는 예약이 이미 마감된 경우가 많다.
- 무료 행사라도 실제로는 사전 신청이 필요하다.
- 주차, 유모차 이동, 수유실 정보가 부족하다.
- 혼잡하면 아이가 힘들어하고 보호자도 피로해진다.
- 날씨가 나쁘면 야외 계획이 바로 취소된다.

**Decision Factors**

- 아이 연령 적합성
- 안전성
- 실내 여부
- 비용
- 혼잡도
- 운영 시간
- 화장실, 수유실, 유모차 접근성
- 대중교통 또는 주차 편의

**Preferred Activities**

- 어린이 박물관
- 도서관 프로그램
- 공공 체험 행사
- 과학관
- 실내 놀이터
- 가족 공연
- 공원 체험 프로그램
- 무료 교육 프로그램

**Weather Sensitivity**

- 매우 높다.
- 비, 폭염, 한파, 미세먼지 모두 외출 결정에 큰 영향을 준다.
- 날씨가 좋더라도 그늘, 휴식 공간, 화장실 접근성을 함께 본다.

**Transportation Preference**

- 자가용 또는 대중교통 모두 사용
- 유모차가 있으면 엘리베이터와 보행 동선 중요
- 주차 정보가 불명확하면 방문 가능성이 낮아진다.
- 이동 시간 40분 이내 선호

### 4. 서울 여행 중인 방문객

**Demographics**

- 연령: 20~50대
- 거주지: 타 지역 또는 해외
- 상황: 서울을 1~5일간 여행 중
- 디지털 사용성: 지도 앱, 번역 앱, 여행 앱, SNS 검색 사용
- 언어: 한국어 사용자 또는 외국어 지원 필요 사용자

**Goals**

- 지금 위치 근처에서 서울다운 경험을 하고 싶다.
- 날씨에 맞는 여행 코스를 빠르게 찾고 싶다.
- 유명하지만 너무 혼잡하지 않은 대안을 알고 싶다.
- 무료 또는 저렴한 공공 문화행사를 발견하고 싶다.

**Frustrations**

- 서울 지리에 익숙하지 않아 동선 판단이 어렵다.
- 유명 관광지는 너무 붐빈다.
- 날씨가 안 좋으면 여행 만족도가 급격히 낮아진다.
- 행사 정보가 한국어로만 제공되어 이해가 어렵다.
- 이동 시간이 실제보다 과소평가되기 쉽다.

**Decision Factors**

- 현재 숙소 또는 현재 위치와의 거리
- 서울만의 특색
- 대중교통 접근성
- 운영 시간
- 비용
- 혼잡도
- 언어 접근성
- 사진/기억에 남는 경험

**Preferred Activities**

- 궁궐, 박물관, 미술관
- 한강
- 전통시장
- 문화행사
- 축제
- 도보 투어
- 야경 명소
- 무료 전시

**Weather Sensitivity**

- 높다.
- 여행 일정이 제한적이므로 날씨가 나빠도 대안을 필요로 한다.
- 비가 오면 실내 관광지, 박물관, 쇼핑·문화공간을 선호한다.

**Transportation Preference**

- 지하철 선호
- 택시는 언어 장벽이나 비용 때문에 보조 수단
- 도보 이동은 관광 경험으로 긍정적이지만 악천후에는 부담
- 환승이 적고 설명이 쉬운 경로 선호

### 5. 돈을 아끼고 싶은 대학생

**Demographics**

- 연령: 19~27세
- 신분: 대학생, 휴학생, 취업준비생
- 거주지: 서울 내 대학가 또는 수도권
- 생활 패턴: 공강, 주말, 저녁 시간에 저비용 활동 탐색
- 디지털 사용성: SNS, 커뮤니티, 지도 앱, 할인 앱 사용 빈도 높음

**Goals**

- 돈을 많이 쓰지 않고 즐길 수 있는 활동을 찾고 싶다.
- 친구나 연인과 부담 없이 갈 곳을 찾고 싶다.
- 무료 행사, 축제, 전시를 놓치지 않고 싶다.
- 교통비까지 고려해 가성비 좋은 선택을 하고 싶다.

**Frustrations**

- 무료 행사 정보가 흩어져 있다.
- “무료”라고 해도 교통비와 식비가 부담된다.
- 인기 있는 무료 행사는 사람이 너무 많다.
- 당일에 갈 수 있는지 확인하기 어렵다.
- 대학가 근처 선택지는 반복적이고 새로움이 부족하다.

**Decision Factors**

- 무료 여부
- 총 예상 비용
- 친구와 함께 가기 좋은지
- 교통비와 이동 시간
- SNS 공유 가능성
- 혼잡도
- 예약 필요 여부

**Preferred Activities**

- 무료 전시
- 축제
- 플리마켓
- 버스킹
- 공공 문화행사
- 대학가 행사
- 한강
- 도서관·문화공간

**Weather Sensitivity**

- 중간~높음.
- 비용이 낮다면 어느 정도 불편은 감수하지만 폭염, 폭우에는 실내를 선호한다.
- 좋은 날씨에는 야외 무료 행사의 선호도가 높다.

**Transportation Preference**

- 지하철, 버스 중심
- 택시 거의 사용하지 않음
- 이동 시간이 길어도 비용이 낮으면 허용
- 막차 시간에 민감

# User Intent Analysis

## 오늘 뭐하지?

사용자가 “오늘 뭐하지?”라고 말할 때 실제로 원하는 것은 단순한 장소 목록이 아니다. 사용자는 지금 남은 시간 안에 실행 가능한 선택지를 원한다. 이 질문에는 결정 피로, 즉시성, 날씨와 이동 부담, 비용 부담이 함께 들어 있다.

실제 해결 과제는 다음과 같다.

- 지금 출발해도 늦지 않은가?
- 너무 멀지 않은가?
- 날씨 때문에 힘들지 않은가?
- 예약 없이 갈 수 있는가?
- 비용이 과하지 않은가?
- 혼잡해서 피곤하지 않은가?
- 혼자 또는 동행자와 가도 괜찮은가?

오늘 추천은 “완벽한 계획”보다 “지금 바로 실행 가능한 좋은 선택”이 중요하다. 사용자는 비교적 짧은 응답, 빠른 판단, 가까운 거리, 명확한 이유를 기대한다.

## 내일 뭐하지?

“내일 뭐하지?”는 오늘보다 계획성이 높다. 사용자는 다음날 하루 또는 반나절을 어떻게 보낼지 미리 정하고 싶어 한다. 내일의 날씨, 운영 시간, 예약 필요 여부, 동행자 일정이 중요하다.

실제 해결 과제는 다음과 같다.

- 내일 날씨에 맞는 활동은 무엇인가?
- 하루 일정으로 무리 없는가?
- 예약이 필요한가?
- 오전/오후/저녁 중 언제 가는 것이 좋은가?
- 동행자와 함께 만족할 만한가?
- 비가 오거나 더우면 대안이 있는가?

내일 추천은 “시간대별 적합성”과 “날씨 예보 기반의 안정성”이 중요하다. 사용자는 당장성보다 실패 확률을 낮추는 추천을 원한다.

## 이번 주말 뭐하지?

“이번 주말 뭐하지?”는 가장 넓은 탐색 의도이다. 사용자는 평일에 하지 못한 여가, 데이트, 가족 외출, 여행성 경험을 찾고 있다. 주말은 시간 여유가 크지만, 동시에 혼잡도와 비용, 날씨 리스크가 커진다.

실제 해결 과제는 다음과 같다.

- 이번 주말 날씨가 좋은 날은 언제인가?
- 토요일과 일요일 중 어느 날이 더 적합한가?
- 서울 전역에서 갈 만한 특별한 행사가 있는가?
- 무료 또는 저비용으로 즐길 수 있는가?
- 사람이 너무 많지는 않은가?
- 하루 코스로 구성할 수 있는가?
- 비가 오면 대체할 수 있는가?

주말 추천은 “발견성”, “코스 가능성”, “무료 여부”, “혼잡 회피”, “플랜 B”가 중요하다. 사용자는 평소보다 더 특별하지만 너무 피곤하지 않은 선택을 원한다.

# Pain Points

1. 날씨, 지도, 행사 정보를 각각 따로 확인해야 한다.
2. 무료 행사 정보가 여러 사이트에 흩어져 있다.
3. “오늘 갈 수 있는지” 운영 시간을 확인하기 어렵다.
4. 행사가 이미 종료되었는데 검색 결과에 남아 있다.
5. 예약이 필요한지 현장에서 알게 되는 경우가 있다.
6. 무료라고 생각했는데 실제로는 유료 체험이나 입장료가 있다.
7. 인기 장소는 도착하면 사람이 너무 많다.
8. 날씨가 갑자기 바뀌면 계획 전체가 무너진다.
9. 블로그 리뷰와 실제 혼잡도가 다르다.
10. 광고성 추천과 실제 추천을 구분하기 어렵다.
11. 현재 위치 기준으로 가까운지 판단하기 어렵다.
12. 대중교통 이동 시간이 예상보다 길다.
13. 환승이 많으면 이동 피로가 커진다.
14. 아이와 갈 수 있는 장소인지 정보가 부족하다.
15. 혼자 가도 괜찮은 분위기인지 알기 어렵다.
16. 데이트 장소가 너무 뻔하고 반복적이다.
17. 주말에는 주차 정보가 부정확하다.
18. 비 오는 날 실내 대안을 다시 검색해야 한다.
19. 폭염이나 한파에 실외 행사를 추천받으면 신뢰가 떨어진다.
20. 가격 정보가 없으면 예산 계획이 어렵다.
21. 같은 장소가 여러 데이터 소스에서 중복 노출된다.
22. 행사 장소명과 지도상의 실제 위치가 다르다.
23. 외국인 방문객은 행사 설명을 이해하기 어렵다.
24. 이동 후 현장에서 휴무를 확인하는 경우가 있다.
25. 추천 이유가 없으면 왜 좋은 선택인지 신뢰하기 어렵다.
26. 너무 많은 선택지가 오히려 결정을 어렵게 만든다.
27. 동행자 취향이 다를 때 중간안을 찾기 어렵다.
28. 날씨가 애매할 때 실내/실외 중 결정하기 어렵다.
29. 교통비까지 포함하면 무료 활동도 비싸게 느껴진다.
30. 일정 공유를 위해 다시 메시지를 정리해야 한다.

# Decision Making Model

## Whether to Go Outside

사용자는 외출 여부를 감정적 기대와 물리적 불편 사이의 균형으로 결정한다. “재미있을 것 같다”는 기대가 “덥다, 멀다, 비 온다, 돈 든다, 사람 많다”는 마찰보다 커야 외출한다.

외출 가능성을 높이는 요인:

- 날씨가 쾌적함
- 이동 시간이 짧음
- 무료 또는 저렴함
- 지금 바로 가능함
- 동행자와 취향이 맞음
- 추천 이유가 명확함
- 대체안이 있음

외출 가능성을 낮추는 요인:

- 폭염, 한파, 비, 높은 습도
- 이동 시간이 길거나 환승이 많음
- 혼잡도가 높음
- 예약 필요
- 가격 불명확
- 운영 시간 불확실
- 정보 신뢰도가 낮음

## How Far to Travel

거리 허용 범위는 시간 의도에 따라 달라진다.

오늘:

- 0~30분 이동이 이상적이다.
- 45분 이상이면 추천 매력이 크게 낮아진다.
- 날씨가 나쁘면 허용 거리가 더 짧아진다.

내일:

- 30~60분 이동도 허용된다.
- 하루 계획이라면 이동 부담을 조금 더 감수한다.
- 단, 예약이나 운영 시간 불확실성이 있으면 멀리 가지 않는다.

이번 주말:

- 특별한 행사라면 60분 이상도 가능하다.
- 무료, 희소성, 동행자 만족도가 높으면 거리를 더 감수한다.
- 하지만 혼잡도와 귀가 편의성이 낮으면 포기 가능성이 높다.

## Whether an Event Is Worth Visiting

사용자는 행사의 가치를 다음 공식처럼 판단한다.

```text
방문 가치 =
  기대 재미
+ 희소성
+ 날씨 적합성
+ 비용 만족도
+ 동행자 적합성
- 이동 피로
- 혼잡 스트레스
- 정보 불확실성
- 예약/대기 부담
```

AI Agent는 이 판단을 대신 구조화해야 한다. 단순히 “추천합니다”가 아니라, “왜 지금 이 조건에서 갈 만한지”를 보여줘야 한다.

## Weather Effects

- 비: 야외 활동 감점, 실내 활동 가중치 상승
- 폭염: 실내, 지하철 접근성, 짧은 도보 동선 중요
- 한파: 실내, 이동 거리, 대기 시간 중요
- 높은 습도: 불쾌지수 상승, 야외 체류 시간 감점
- 맑고 선선함: 야외 축제, 공원, 한강, 도보 코스 가중치 상승

## Cost Sensitivity

비용 민감도는 사용자별로 다르다.

- 대학생: 무료 여부와 교통비에 매우 민감
- 직장인: 1~2만원까지 허용 가능하나 가성비를 중시
- 커플: 분위기와 만족도가 높으면 비용 허용 범위 증가
- 가족: 인원수 때문에 총비용에 민감
- 여행객: 서울다운 경험이라면 비용 허용 범위가 넓어짐

## Crowd Sensitivity

혼잡도는 단순 불편 요소가 아니라 추천 신뢰를 좌우한다.

- 혼자 활동하는 사용자는 조용함을 선호한다.
- 커플은 너무 붐비면 대화와 분위기가 깨진다.
- 가족은 혼잡하면 안전과 피로 문제가 커진다.
- 여행객은 유명 장소의 혼잡을 어느 정도 감수하지만 대체안을 원한다.
- 대학생은 인기 무료 행사를 선호하지만 대기 시간이 길면 이탈한다.

## Travel Friction

이동 마찰은 거리보다 더 복합적이다.

- 환승 횟수
- 역에서 도보 거리
- 날씨에 노출되는 시간
- 막차 시간
- 주차 가능성
- 유모차/휠체어 접근성
- 동행자와 만나는 중간 지점
- 귀가 동선

AI Agent는 단순 직선거리보다 “실제 가기 편한가”를 우선해야 한다.

# Recommendation Expectations

## Relevance

사용자는 자신의 현재 맥락에 맞는 추천을 기대한다. 같은 장소라도 폭염, 비, 주말 혼잡, 동행자에 따라 적합도가 달라져야 한다.

좋은 추천:

- 현재 위치와 가까움
- 날씨에 맞음
- 시간대에 맞음
- 비용 조건에 맞음
- 동행자와 맞음
- 실내/실외 선호에 맞음

나쁜 추천:

- 이미 종료된 행사
- 현재 갈 수 없는 장소
- 폭염에 야외 장시간 활동
- 비 오는 날 노출이 많은 코스
- 무료 요청에 유료 장소
- 혼자 가기 어색한 장소

## Convenience

사용자는 검색을 줄이고 싶어 한다. 추천은 바로 행동으로 이어질 수 있어야 한다.

필수 정보:

- 어디인지
- 지금 갈 수 있는지
- 얼마나 걸리는지
- 얼마인지
- 왜 추천하는지
- 혼잡한지
- 비 오면 대안이 있는지

## Trustworthiness

추천 신뢰는 데이터 근거에서 나온다.

신뢰를 높이는 요소:

- 데이터 기준 시각 표시
- 출처 표시
- 운영 시간 확인
- 가격 정보 명확화
- 혼잡도 근거 표시
- 불확실한 정보는 불확실하다고 말하기
- 추천 이유가 실제 데이터와 일치하기

## Diversity

사용자는 같은 유형만 반복되는 추천을 싫어한다. 단, 다양성은 관련성을 해치지 않아야 한다.

권장 구성:

- 가장 가까운 선택지 1개
- 날씨에 가장 적합한 선택지 1개
- 무료/가성비 선택지 1개
- 특별한 경험 선택지 1개
- 플랜 B 1개

## Timing

추천은 시간 맥락을 반영해야 한다.

오늘:

- 즉시 출발 가능
- 현재 운영 중
- 예약 필요 최소화
- 이동 시간 짧음

내일:

- 시간대별 예보 반영
- 하루 계획 가능
- 운영 시간 여유
- 예약 여부 명확

주말:

- 토/일 날씨 비교
- 혼잡 가능성 반영
- 무료 행사 우선 탐색
- 대체안 포함

# Behavioral Insights

1. 사용자는 “최고의 선택”보다 “실패하지 않는 선택”을 더 자주 원한다.
2. 선택지가 너무 많으면 결정 피로가 커지므로 3~5개 추천이 적절하다.
3. 추천 이유가 명확하면 낯선 장소도 선택할 가능성이 높아진다.
4. 무료 여부는 비용 이상의 심리적 안전감을 준다.
5. 날씨가 나쁘면 사용자는 탐험보다 안정성을 선호한다.
6. 혼잡도는 현장 만족도를 크게 좌우하지만 사전에 확인하기 어렵다.
7. 사용자는 이동 시간을 실제 비용처럼 느낀다.
8. “예약 필요”는 당일 추천에서 큰 장벽이다.
9. 커플은 객관적 최적보다 두 사람 모두 무난히 만족할 선택을 선호한다.
10. 가족 사용자는 재미보다 안전, 편의시설, 피로도를 우선한다.
11. 대학생은 무료 활동이라도 교통비와 식비까지 포함해 판단한다.
12. 여행객은 “서울다운 경험”이라는 상징성을 중요하게 본다.
13. 혼자 활동하는 사용자는 분위기 적합성과 심리적 편안함을 중요하게 본다.
14. 사용자는 날씨 앱 수치를 해석하기보다 “그래서 뭘 하면 좋은지”를 원한다.
15. 비 예보가 애매하면 실내/실외 혼합 추천이 신뢰를 높인다.
16. 사용자는 직접 조건을 모두 말하지 않으므로 Agent가 맥락을 추론해야 한다.
17. 사용자가 후속 요청을 할 때는 기존 추천의 방향은 유지하되 조건만 조정하길 기대한다.
18. 추천 결과의 첫 번째 카드는 가장 안전한 선택이어야 한다.
19. 두 번째나 세 번째 카드는 발견성과 다양성을 제공할 수 있다.
20. “데이터 기준 시각”은 추천의 신뢰도를 높이는 작은 장치다.

# Edge Cases

1. 비가 예상보다 일찍 시작된다.
2. 갑자기 폭우가 내린다.
3. 강수 확률은 낮지만 현장에서 소나기가 온다.
4. 행사 당일 취소가 발생한다.
5. 행사 장소가 변경된다.
6. 행사 시간이 잘못 등록되어 있다.
7. 무료 행사지만 사전 예약이 필요하다.
8. 예약이 이미 마감되었다.
9. 현장 대기줄이 매우 길다.
10. 장소가 갑자기 임시 휴관한다.
11. 운영 시간 정보가 데이터 소스마다 다르다.
12. 위치 좌표가 잘못되어 있다.
13. 같은 행사가 중복 데이터로 들어온다.
14. 가격 정보가 비어 있다.
15. 무료 여부가 “부분 무료”이다.

# Success Metrics

## User Success Metrics

### Recommendation Acceptance Rate

사용자가 추천 결과 중 하나를 클릭하거나 상세 보기, 지도 보기, 공유, 일정 저장 등 후속 행동을 수행한 비율.

목표:

- MVP 기준 35% 이상
- 개선 후 50% 이상

### Time to Decision

사용자가 질문 입력 후 추천 중 하나를 선택하기까지 걸린 시간.

목표:

- 오늘 추천: 2분 이내
- 내일 추천: 5분 이내
- 주말 추천: 7분 이내

### Action Completion Rate

추천 이후 실제 행동으로 이어지는 액션의 완료율.

측정 이벤트:

- 지도 보기
- 일정 저장
- 공유
- 플랜 B 확인
- 재추천 요청

### Relevance Satisfaction

사용자가 추천이 자신의 상황에 맞는다고 평가한 비율.

측정 방법:

- “이 추천이 도움이 되었나요?” 피드백
- 추천 카드별 좋아요/싫어요
- 후속 조건 변경 빈도

### Trust Score

사용자가 추천을 신뢰할 수 있다고 느끼는 정도.

신뢰를 높이는 지표:

- 추천 이유 열람률
- 데이터 기준 시각 노출 후 이탈률 감소
- 출처 링크 클릭률
- “정보가 틀림” 신고율 감소

### Plan B Usage Rate

비, 폭염, 혼잡 등 리스크 상황에서 대체 추천을 사용한 비율.

의미:

- Agentic 가치가 실제로 전달되는지 확인
- 단순 추천 서비스와의 차별점 측정

### Repeat Usage

사용자가 다시 “오늘/내일/주말 뭐하지?”를 입력하는 빈도.

목표:

- 7일 내 재사용률
- 30일 내 재사용률
- 주말 전 재방문율

## Business Success Metrics

### Weekly Active Users

주간 활성 사용자 수. 특히 금요일~일요일 사용량을 핵심으로 본다.

### Query Volume by Intent

의도별 질문량.

- today
- tomorrow
- weekend
- free
- indoor
- date
- family
- solo

이를 통해 제품의 핵심 사용 맥락을 파악한다.

### Recommendation Engagement

추천 카드와 상호작용한 비율.

측정 이벤트:

- 카드 클릭
- 상세 열기
- 지도 보기
- 공유
- 저장
- 다른 조건으로 재추천

### Data Coverage Rate

추천 후보 중 필수 데이터가 채워진 비율.

필수 데이터:

- 위치
- 운영 기간
- 비용
- 실내/실외
- 거리
- 날씨 적합도
- 혼잡도 또는 혼잡도 confidence

### API Reliability

MCP Tool별 성공률과 응답 시간.

목표:

- Weather Tool 성공률 99% 이상
- Culture Event Tool 성공률 98% 이상
- Congestion Tool 성공률 95% 이상
- Recommendation Tool 성공률 99.5% 이상

### Fallback Success Rate

외부 API 실패 시에도 부분 추천을 성공적으로 제공한 비율.

목표:

- API 일부 실패 상황에서 사용자 응답 성공률 90% 이상

### Share Conversion

추천 결과가 공유된 횟수와 공유 링크를 받은 사람이 다시 방문한 비율.

의미:

- 데이트, 가족, 친구 활동에서 제품 확산 가능성 측정

### Retention by Persona

주요 페르소나별 재사용률.

- 혼자 활동 사용자
- 커플
- 가족
- 여행객
- 대학생

페르소나별 retention 차이는 추천 품질과 UX 개선 우선순위를 결정한다.

### Recommendation Diversity Health

특정 장소나 카테고리에 추천이 과도하게 쏠리지 않는지 측정한다.

지표:

- 상위 10개 장소 추천 집중도
- 실내/실외 추천 비율
- 무료/유료 추천 비율
- 자치구별 추천 분포
- 카테고리별 추천 분포

### Complaint and Correction Rate

사용자가 “정보가 틀렸다”, “운영하지 않았다”, “너무 멀다”, “무료가 아니었다” 등으로 신고한 비율.

목표:

- 전체 추천 대비 2% 이하
- 치명적 오류는 0.5% 이하

## UX Decision Principles for AI Agents

1. 사용자의 질문이 짧을수록 Agent가 맥락을 더 적극적으로 보완해야 한다.
2. 오늘 추천은 가까움과 즉시성을 최우선으로 한다.
3. 내일 추천은 날씨와 시간대 적합성을 최우선으로 한다.
4. 주말 추천은 날씨, 무료 여부, 발견성, 혼잡도를 균형 있게 본다.
5. 폭염, 비, 한파, 미세먼지 상황에서는 실내 추천을 우선한다.
6. 사용자가 무료를 언급하면 가격 정보가 없는 후보를 무료로 간주하지 않는다.
7. 혼잡도 정보가 없으면 “확인 불가”라고 명시하고 기본 점수로 처리한다.
8. 추천은 최대 5개를 넘기지 않는다.
9. 첫 번째 추천은 가장 안전하고 실패 확률이 낮아야 한다.
10. 추천 이유는 반드시 실제 데이터 기반이어야 한다.
11. 사용자의 후속 요청은 기존 intent를 유지한 채 조건만 조정한다.
12. 불확실성이 높으면 플랜 B를 함께 제안한다.
